Rosen 1.1 #1-4 — Truth Tables▾

#1: Which of these are propositions? (a) Boston is capital of MA. (b) Miami is capital of FL. (c) 2+3=5. (d) 5+7=10. (e) x+2=11. (f) Answer this question.

#3: Truth table for ¬p ∧ q.

#4: Truth table for ¬(p ∨ q).

Answers

Step 1 (#1): (a) "Boston is capital of MA" — declarative, truth value T → proposition. (b) "Miami is capital of FL" — declarative, truth value F (Tallahassee) → proposition. (c) "2+3=5" — declarative, T → proposition. (d) "5+7=10" — declarative, F → proposition. (e) "x+2=11" — truth depends on x → not a proposition. (f) "Answer this question" — command → not a proposition.

Step 2 (#3): ¬p ∧ q is true only when p = F and q = T. For all other combinations of p,q it is false.

Step 3 (#4): ¬(p ∨ q) is true only when p = F and q = F. This is equivalent to ¬p ∧ ¬q by De Morgan's law.

Rosen 1.1 #18 — Conditional Equivalence▾

Are (p → q) ∧ (p → r) and p → (q ∧ r) logically equivalent?

Yes

Step 1: Rewrite (p→q) ∧ (p→r) using the implication equivalence: (¬p ∨ q) ∧ (¬p ∨ r).

Step 2: Factor out ¬p using distributivity: (¬p ∨ q) ∧ (¬p ∨ r) ≡ ¬p ∨ (q ∧ r).

Step 3: Convert back to implication: ¬p ∨ (q ∧ r) ≡ p → (q ∧ r).

Step 4: Both are false only when p = T and at least one of q,r is F. Same truth table → logically equivalent.

Key Insight: The equivalence shows that "if p then q, and if p then r" is the same as "if p then (q and r)" which makes intuitive sense — if p forces both q and r individually, it forces them together.

Rosen 1.1 #24 — Contrapositive▾

Show p → q and ¬q → ¬p are equivalent without a truth table.

Solution

Step 1: Rewrite p → q using the implication equivalence: p → q ≡ ¬p ∨ q.

Step 2: Rewrite ¬q → ¬p using the same rule: ¬q → ¬p ≡ ¬(¬q) ∨ ¬p ≡ q ∨ ¬p.

Step 3: By commutativity of ∨: q ∨ ¬p ≡ ¬p ∨ q.

Step 4: Both expressions reduce to ¬p ∨ q, so they are logically equivalent.

Key Insight: The contrapositive has the same truth value as the original implication. This is not true for the converse (q → p) or inverse (¬p → ¬q).

Common Mistake

Mistake: Confusing contrapositive with converse (q → p) or inverse (¬p → ¬q).

Why it matters: The contrapositive is equivalent to the original, but the converse and inverse are not — they may have different truth values.

Rosen 1.1 #33 — Bitwise Operations▾

Find bitwise OR, AND, XOR of:
(a) 101 1110, 010 0001
(b) 1111 0000, 1010 1010

Answers

Step 1 (a) OR: Align bits: 1011110 | 0100001 = 1111111 (bit-by-bit logical OR).

Step 2 (a) AND: 1011110 & 0100001 = 0000000 (bit-by-bit logical AND, no overlapping 1s).

Step 3 (a) XOR: 1011110 ⊕ 0100001 = 1111111 (bit-by-bit XOR, each position differs).

Step 4 (b) OR: 11110000 | 10101010 = 11111010.

Step 5 (b) AND: 11110000 & 10101010 = 10100000.

Step 6 (b) XOR: 11110000 ⊕ 10101010 = 01011010.

Rosen 1.1 #41 — 3-Var Compound▾

Truth table for (p → q) → (q → p).

Key Insight

Step 1: This is NOT a tautology. To see why, consider the case p = F, q = T.

Step 2: When p = F, q = T: (p → q) = (F → T) = T. But (q → p) = (T → F) = F.

Step 3: So T → F = F, which means the whole compound is false.

Step 4: This proves that the converse (q → p) is not equivalent to the original implication (p → q).

Rosen 1.1 #46 — Program Conditionals▾

Value of x after each if x=1 before?
(a) if (x+2=3) then x:=x+1
(b) if (x+1=3) OR (2x+2=3) then x:=x+1
(c) if (2x+3=5) AND (3x+4=7) then x:=x+1
(d) if (x+1=2) XOR (x+2=3) then x:=x+1

Answers

Step 1 (a): x+2 = 1+2 = 3, so condition is true. x becomes 1+1 = 2.

Step 2 (b): x+1 = 2 ≠ 3 (false). 2x+2 = 4 ≠ 3 (false). false OR false = false. x stays 1.

Step 3 (c): 2x+3 = 5 (true). 3x+4 = 7 (true). true AND true = true. x becomes 2.

Step 4 (d): x+1 = 2 (true). x+2 = 3 (true). true XOR true = false. x stays 1.

Rosen 1.1 #52 — Liar Paradox▾

Is "This statement is false" a proposition?

No.

Step 1: Assume the statement is true. Then what it says ("this statement is false") must hold, so it is false. Contradiction.

Step 2: Assume the statement is false. Then the negation of "this statement is false" is "this statement is true", so it is true. Contradiction.

Step 3: Since neither truth value yields a consistent assignment, the statement is not a proposition.

Key Insight: Self-referential statements can break the fundamental requirement of propositional logic: every proposition must be either true or false, not both and not neither.

Rosen 1.3 #35 — NAND Universality▾

Express ¬p, p∧q, p∨q, p→q using only the NAND operator (p|q ≡ ¬(p∧q)).

Solution

Step 1 (¬p): ¬p ≡ p|p. Since p|p = ¬(p∧p) = ¬p.

Step 2 (p∧q): p∧q ≡ (p|q)|(p|q). First p|q = ¬(p∧q). Then (p|q)|(p|q) = ¬(¬(p∧q) ∧ ¬(p∧q)) = ¬¬(p∧q) = p∧q.

Step 3 (p∨q): p∨q ≡ (p|p)|(q|q) = ¬p|¬q = ¬(¬p ∧ ¬q) = p ∨ q (by De Morgan).

Step 4 (p→q): p→q ≡ ¬p∨q ≡ (p|p)|(q|q), using the OR expression from Step 3 with ¬p and q.

Key Insight: Since we can build ¬, ∧, and ∨ from NAND, any logical operator can be expressed using only NAND — making NAND a universal gate.

Common Mistake

Mistake: Forgetting that NAND is not associative: (p|q)|r ≠ p|(q|r) in general.

Why it matters: When building complex expressions with NAND, the grouping (parentheses) is crucial — you can't rearrange them freely like AND or OR.

Rosen 1.4 #28 — Predicate Translation▾

Domain = people at a university. Let S(x): x is a student, E(x): x is enrolled, C(x): x is a course, T(x,y): x is taking y. Translate:

  1. Every student is enrolled.
  2. Some student is not enrolled.
  3. Every student is taking some course.
  4. Some course is being taken by every student.
Answers

Step 1 (1): "Every student is enrolled" = ∀x (S(x) → E(x)). If someone is a student, they are enrolled. Not ∀x (S(x) ∧ E(x)) — that would say everyone is a student and enrolled.

Step 2 (2): "Some student is not enrolled" = ∃x (S(x) ∧ ¬E(x)). There exists at least one person who is both a student and not enrolled.

Step 3 (3): "Every student is taking some course" = ∀x (S(x) → ∃y (C(y) ∧ T(x,y))). For each student, there exists some course that they take.

Step 4 (4): "Some course is being taken by every student" = ∃x (C(x) ∧ ∀y (S(y) → T(y,x))). There exists a course that every student takes.

Key Insight: Statements (3) and (4) differ in quantifier order — in (3) each student can take a different course, but in (4) there must be one common course for all students. The quantifier swap changes the meaning entirely.

Common Mistake

Mistake: Translating "every student is taking some course" as ∀x∃y or ∃y∀x without understanding the difference.

Why it fails: ∀x∃y means "each student has some course (possibly different)" while ∃y∀x means "there is one course that all students take." These are not equivalent.

Rosen 1.5 #12 — Nested Quantifier Order▾

Let domain = ℝ. Are ∀x ∃y (x + y = 0) and ∃y ∀x (x + y = 0) logically equivalent?

No

Step 1: Evaluate ∀x ∃y (x + y = 0). For each x, we can pick y = −x. Since every real number has an additive inverse, this is always true.

Step 2: Evaluate ∃y ∀x (x + y = 0). This asks: does there exist a single y that works for all x simultaneously? If y = 0, then x+0=0 fails for x=1. If y = −1, it fails for x=0. No single y can satisfy all x.

Step 3: Since one is true and the other is false, they are not logically equivalent.

Key Insight: ∀x∃y says "each x has its own y" while ∃y∀x says "one y works for all x." The latter is a much stronger claim.

Common Mistake

Mistake: Assuming ∀x∃y and ∃y∀x are always interchangeable.

Why it fails: ∃y∀x is much stronger — it requires a single y that simultaneously satisfies the predicate for every x. Only certain symmetric predicates (like x+y = y+x) allow this.

M4CS §3 — Textbook Problems▾
M4CS 3.1 — Logical Equivalence Tautology▾

Show (P → Q) ∨ (Q → P) is a tautology.

Proof

Step 1: We show that regardless of the truth values of P and Q, at least one disjunct is always true.

Step 2: Consider P. Either P is true or P is false.

Step 3 (P true): If P is true, then Q → P is true because the consequent (P) is true. So the disjunction (P→Q) ∨ (Q→P) is true.

Step 4 (P false): If P is false, then P → Q is true because the antecedent (P) is false. So the disjunction is true.

Step 5: In both cases, the compound is true. Therefore it is a tautology.

Key Insight: This tautology shows that for any two propositions, at least one direction of implication always holds — even if P and Q are unrelated. This is sometimes called "implication tautology."

M4CS 3.6 — Predicate Logic (Monkey/TA/Pizza)▾

Domain = students. Translate: "If a student is a TA, then they have a pizza or a monkey." Use TA(x), Pizza(x), Monkey(x).

Solution

Step 1: "If a student is a TA" — this is conditional, so we use →. The domain is all students, so we quantify with ∀.

Step 2: "then they have a pizza or a monkey" — the consequent is Pizza(x) ∨ Monkey(x).

Step 3: Combine: ∀x (TA(x) → (Pizza(x) ∨ Monkey(x))).

Key Insight: "If...then" translates to →, not ∧. Using ∧ would claim that every student is a TA.

Common Mistake

Mistake: Writing ∀x (TA(x) ∧ (Pizza(x) ∨ Monkey(x))).

Why it fails: This says every student is a TA and (has pizza or monkey), which is much stronger than the original. The correct form uses → to restrict the claim only to TAs.

M4CS 3.26 — Quantified Truth▾

Determine the truth value of ∃x ∈ ℤ: x² − x − 1 > 0 and ∀x ∈ ℤ: x² − x − 1 > 0.

Answers

Step 1: For ∃x (x² − x − 1 > 0): pick x = 2. Then 4 − 2 − 1 = 1 > 0. At least one integer satisfies it → true.

Step 2: For ∀x (x² − x − 1 > 0): pick x = 0. Then 0 − 0 − 1 = −1 > 0 is false. One counterexample kills the universal claim → false.

Key Insight: One witness proves an existential claim; one counterexample disproves a universal claim.

Common Mistake

Mistake: Confusing the roles of witnesses vs. counterexamples for existential and universal quantifiers.

Why it fails: A single example does not prove a universal claim, and a single failure does not disprove an existential claim. They are duals of each other — De Morgan's laws for quantifiers.

M4CS 3.33 — Validity of Quantified Implication▾

Show (∀x P(x)) ⇒ (∃x P(x)) is valid when the domain is nonempty.

Proof

Step 1: Assume ∀x P(x) is true — every element in the domain satisfies P(x).

Step 2: Since the domain is nonempty, there exists at least one element in it.

Step 3: Pick any such element. By the assumption, that element satisfies P(x).

Step 4: Therefore ∃x P(x) holds. Since the implication always holds, it is valid.

Key Insight: This inference requires a nonempty domain. If the domain were empty, ∀x P(x) would be vacuously true but ∃x P(x) would be false — breaking the implication.

MIT A01 — Predicate Logic Translation▾

Domain = people. F(x): Ferrari, H(x): fast, C(x): car, R(x): race car driver. Translate:

  1. All Ferraris are fast.
  2. Some cars are Ferraris.
  3. No race car driver is slow.
  4. All race car drivers own a fast car.
Answers

Step 1 (1): "All Ferraris are fast" = ∀x (F(x) → H(x)). Being a Ferrari implies being fast.

Step 2 (2): "Some cars are Ferraris" = ∃x (C(x) ∧ F(x)). There exists something that is both a car and a Ferrari.

Step 3 (3): "No race car driver is slow" = ∀x (R(x) → H(x)). Being a race car driver implies being fast (not slow). Alternatively, ¬∃x (R(x) ∧ ¬H(x)).

Step 4 (4): "All race car drivers own a fast car" = ∀x (R(x) → ∃y (C(y) ∧ H(y) ∧ owns(x,y))). For each race car driver, there exists a car that is fast and they own it.

Common Mistake

Mistake: Writing "All As are Bs" as ∀x (A(x) ∧ B(x)).

Why it fails: ∀x (A(x) ∧ B(x)) says everything is both A and B, which is clearly not what "all As are Bs" means. The correct form is ∀x (A(x) → B(x)).

MIT A01 — NAND Universality▾

Express ¬p, p∧q, p∨q, p→q using only NAND.

Solution

Step 1 (¬p): p|p = ¬(p∧p) = ¬p.

Step 2 (p∧q): First compute p|q = ¬(p∧q). Then (p|q)|(p|q) = ¬(¬(p∧q)) = p∧q.

Step 3 (p∨q): Compute ¬p = p|p and ¬q = q|q. Then (p|p)|(q|q) = ¬(¬p ∧ ¬q) = p ∨ q (De Morgan).

Step 4 (p→q): p→q = ¬p ∨ q = (p|p)|(q|q), using the OR construction from Step 3.

Domain = wizards. S:Spy, D:Death Eater, O:Order of Phoenix, G:Gryffindor. Translate.

Answers

2. Proof Techniques Set 2

Direct & Contrapositive Proofs▾
Direct: To prove P ⇒ Q, assume P and deduce Q.
Contrapositive: Prove ¬Q ⇒ ¬P instead.

Example: n² even ⇒ n even. Prove contrapositive: if n odd, n² odd.

Proof by Contradiction▾
Assume ¬P, derive contradiction (R ∧ ¬R). Conclude P.

Classic — √2 irrational: Assume √2 = a/b in lowest terms. Then a²=2b², so a even, a=2c. Then b²=2c², b even. Contradiction.

Proof by Cases▾
Exhaustive cases C₁∨...∨Cₖ, prove each Cᵢ⇒P.

Example: n²+n even for all n. Cases: n even (2k) or n odd (2k+1). Both give n²+n=2(...).

Proof Problems▾
DM A1 Q4 — Contrapositive▾

Prove: n irrational ⇒ n^{1/3} irrational.

Contrapositive

Step 1: We prove the contrapositive: if n^{1/3} is rational, then n is rational.

Step 2: Assume n^{1/3} = p/q where p,q are integers with q ≠ 0 in lowest terms.

Step 3: Cube both sides: n = p³/q³.

Step 4: Since p³ and q³ are integers, n = p³/q³ is rational.

Step 5: The contrapositive is proved, so the original statement holds.

DM A2 Q1 — Cases: Email▾

Alice→Bob (uni), Bob→Charlie, Charlie not uni. Prove one email uni→personal.

Cases on Bob's email

Step 1: We know Alice sent to Bob using a university account. Charlie did not use a university account. Bob sent some email to Charlie.

Step 2 (Case 1): Bob used a university account for the email to Charlie. Then Bob→Charlie is university → personal (since Charlie didn't use university).

Step 3 (Case 2): Bob used a personal account for the email to Charlie. Then Alice→Bob is university → personal (since Bob used personal).

Step 4: In either case, at least one email in the chain is from a university account to a personal account.

MIT A01 — Fake Coin (12 coins)▾

12 coins, one counterfeit. Find it in 3 weighings, determine heavy/light.

Strategy (3³=27 > 24)

Step 1: Weigh 1: 4 coins vs 4 coins. Three outcomes possible (left heavy, balanced, right heavy). 3 weighings give 3³ = 27 outcomes, enough to distinguish 12×2 = 24 possibilities (12 coins × heavy/light).

Step 2 (Case balanced): The fake is among the 4 aside coins. Weigh 2: 3 known-good coins vs 3 suspicious coins. This narrows it down.

Step 3 (Case unbalanced): Label the 4 heavier-side coins H and 4 lighter-side coins L. Weigh 2 recombines coins strategically (e.g., 3H+1L vs 1H+3L+1good) to narrow candidates.

Step 4: Weigh 3 resolves the final 2-3 candidate coins by comparing against a known-good coin.

Key Insight: The information-theoretic bound: 3³ = 27 must be ≥ 24 (12 coins × 2 possibilities each). If it were 13 coins (26 possibilities) it would still be possible, but 14 (28) would not.

Hard — Friends & Strangers▾

In any group of 6, 3 mutual friends or 3 mutual strangers.

Pigeonhole + Cases

Step 1: Pick any person p. Among the other 5 people, by the pigeonhole principle, p has either ≥3 friends or ≥3 strangers (since 5 = 3+2).

Step 2 (Subcase: ≥3 friends of p): If any two of those friends are friends with each other, then together with p they form 3 mutual friends. If none of them are friends, they are all mutual strangers — giving 3 mutual strangers.

Step 3 (Subcase: ≥3 strangers to p): Symmetric argument — if any two of those strangers are strangers to each other, then p + those two form 3 mutual strangers. Otherwise they are all mutual friends.

Step 4: In all cases, we find either 3 mutual friends or 3 mutual strangers. This is a Ramsey number R(3,3) = 6.

Key Insight: This is a classic application of the pigeonhole principle combined with case analysis — known as Ramsey theory.

Medium — Birthday Paradox Flaw▾

Find flaw: "Proof" all people share the same birthday.

Step fails at n=1→2

Step 1: The faulty proof attempts induction: assume any group of n people share the same birthday. For n+1, consider the first n and the last n — by IH each group shares a birthday, and since the groups overlap, all share the same birthday.

Step 2: The flaw: when n = 1 (going from 1 to 2 people), the "first n" = {person 1} and the "last n" = {person 2} are disjoint — there is no overlapping person to bridge the claim.

Step 3: The argument works only for n ≥ 2 where the groups have at least one person in common. The inductive step fails at the base transition from n=1 to n=2.

M4CS 1.7 #8 Irrational^Rational▾

Prove there exist irrational numbers a,b such that a^b is rational. (Hint: Consider √2^√2.)

Non-Constructive Proof

Step 1: Consider the number √2^√2. It is either rational or irrational.

Step 2 (Case 1): If √2^√2 is rational, then we are done — take a = √2 and b = √2, and a^b is rational.

Step 3 (Case 2): If √2^√2 is irrational, let a = √2^√2 (irrational) and b = √2 (irrational). Then a^b = (√2^√2)^√2 = √2^(√2·√2) = √2^2 = 2, which is rational.

Step 4: In either case, we have found irrational a,b with a^b rational.

Key Insight: Non-constructive proof — we prove existence without knowing which case actually holds.

Common Mistake

Mistake: Trying to construct a and b explicitly from the start instead of using a case analysis.

Why it fails: We don't know whether √2^√2 is rational or irrational, so any explicit construction would require resolving an open problem. The case split avoids this.

M4CS 1.8 #16 √2 + √3 irrational▾

Prove √2 + √3 is irrational.

Proof by Contradiction

Step 1: Assume for contradiction that √2 + √3 is rational. Then √2 + √3 = p/q for integers p,q with q ≠ 0 in lowest terms.

Step 2: Square both sides: (√2 + √3)^2 = 2 + 3 + 2√6 = 5 + 2√6 = p^2/q^2.

Step 3: Isolate √6: 2√6 = p^2/q^2 - 5, so √6 = (p^2/q^2 - 5)/2.

Step 4: The left side (√6) is irrational. The right side is a rational combination of integers and thus rational.

Step 5: This is a contradiction — an irrational number cannot equal a rational number. Therefore our assumption was false, and √2 + √3 is irrational.

Key Insight: Square the sum to isolate √6, then use the closure of rational numbers under arithmetic operations to reach a contradiction.

Rosen 1.7 #6 Direct Proof▾

Prove the product of two odd integers is odd.

Direct Proof

Step 1: Let a and b be arbitrary odd integers. By definition of odd, a = 2k + 1 and b = 2m + 1 for some integers k,m.

Step 2: Compute the product: a·b = (2k + 1)(2m + 1) = 4km + 2k + 2m + 1.

Step 3: Factor out a 2 from the first three terms: 4km + 2k + 2m = 2(2km + k + m).

Step 4: So a·b = 2(2km + k + m) + 1, which is of the form 2·(integer) + 1.

Step 5: Therefore a·b is odd by definition, completing the proof.

Common Mistake

Mistake: Showing the sum instead of the product. The sum of two odd numbers is even (2k+1 + 2m+1 = 2(k+m+1)), but the product is odd. Don't confuse the two.

Rosen 1.7 #16 Contrapositive▾

Prove by contrapositive: if x+y ≥ 2, then x ≥ 1 or y ≥ 1.

Contrapositive

Step 1: The contrapositive of "P → Q" is "¬Q → ¬P". Here P is "x+y ≥ 2" and Q is "x ≥ 1 or y ≥ 1".

Step 2: Negate Q: ¬(x ≥ 1 or y ≥ 1) means x < 1 and y < 1 by De Morgan's law.

Step 3: Negate P: ¬(x+y ≥ 2) means x+y < 2.

Step 4: So the contrapositive is: if x < 1 and y < 1, then x+y < 2.

Step 5: This is true because the sum of two numbers each strictly less than 1 is strictly less than 2.

Key Insight: The contrapositive is logically equivalent to the original statement, so proving it suffices. This approach avoids case analysis on whether x or y meets the threshold.

Rosen 1.8 #1 Proof by Cases▾

Prove n² + 1 ≥ 2n for all integers n.

Case Analysis

Step 1: We consider two cases: n ≥ 1 and n ≤ 0. These cover all integers.

Step 2 (Case n ≥ 1): (n−1)² ≥ 0 for all integers. Expand: n² − 2n + 1 ≥ 0, so n² + 1 ≥ 2n.

Step 3 (Case n ≤ 0): n² ≥ 0 always. Since n ≤ 0, we have 2n ≤ 0, so 1 ≥ 2n. Adding n² (which is ≥ 0) to both sides: n² + 1 ≥ 2n.

Step 4: Both cases give n² + 1 ≥ 2n, so the statement holds for all integers n.

Key Insight: The (n−1)² approach works only for n ≥ 1 (since it relies on real-number squaring). For n ≤ 0 we need a separate argument.

Common Mistake

Mistake: Only proving for positive n using the (n−1)² method and forgetting the n ≤ 0 case.

Why it fails: Without the second case, the proof is incomplete — the claim must hold for all integers.

M4CS 1.8 #12 Contradiction▾

Prove if a·b ≤ n then a ≤ √n or b ≤ √n for positive integers a,b.

Contrapositive

Step 1: Prove the contrapositive: if a > √n and b > √n, then a·b > n.

Step 2: Since a,b are positive integers, we can multiply inequalities: a > √n and b > √n implies a·b > √n·√n = n.

Step 3: This contradicts a·b ≤ n, so the original implication holds.

Key Insight: The contrapositive swaps the conclusion inequality into a single multiplication, which is much simpler to reason about.

DM A1 Q3 — Set Equality▾

Prove: A = (A−B) ∪ (A∩B).

Two-direction proof

Step 1 (⊆ direction): Take x ∈ A. Either x ∈ B or x ∉ B.

Step 2: If x ∈ B, then x ∈ A∩B. If x ∉ B, then x ∈ A−B.

Step 3: In either case, x ∈ (A−B) ∪ (A∩B). So A ⊆ (A−B) ∪ (A∩B).

Step 4 (⊇ direction): Take x ∈ (A−B) ∪ (A∩B).

Step 5: If x ∈ A−B, then x ∈ A. If x ∈ A∩B, then x ∈ A.

Step 6: In either case, x ∈ A. So (A−B) ∪ (A∩B) ⊆ A.

Step 7: Both directions hold, therefore A = (A−B) ∪ (A∩B). ✓

Common Mistake

Mistake: Only proving one direction (usually ⊆).

Why it fails: Set equality requires both A ⊆ B and B ⊆ A. Proving only one direction only shows that A is a subset of B, not that they are equal.

M4CS 4.1 #3 Set Formula to Propositional▾

Convert set identity (A−B)∪(A∩B) = A into a propositional equivalence and verify.

Solution

Step 1: Translate membership in the LHS to propositional logic: x ∈ (A−B)∪(A∩B) ≡ (x∈A ∧ x∉B) ∨ (x∈A ∧ x∈B).

Step 2: Factor out x∈A using distributivity: (x∈A ∧ x∉B) ∨ (x∈A ∧ x∈B) ≡ x∈A ∧ (x∉B ∨ x∈B).

Step 3: The inner clause (x∉B ∨ x∈B) is a tautology (always true). So we get x∈A ∧ T.

Step 4: x∈A ∧ T ≡ x∈A, which is exactly the RHS. So the identity holds.

Common Mistake

Mistake: Not properly converting set operations to propositional connectives.

Why it fails: Set operations (∪, ∩, −) have direct propositional analogs (∨, ∧, ∧¬). Skipping this conversion leads to algebraic errors.

Step 3: List subsets by size: ∅; {a}, {b}, {c}; {a,b}, {a,c}, {b,c}; {a,b,c}.

Step 4: P({a,b,c}) = {∅, {a}, {b}, {c}, {a,b}, {a,c}, {b,c}, {a,b,c}}.

M4CS 4.1 #5 De Morgan for Sets▾

Prove A∩B = A̅ ∪ B̅ using propositional logic.

Solution

Step 1: Translate the complement of intersection: x ∈ A∩B means x ∉ (A∩B).

Step 2: x ∈ A∩B ≡ ¬(x ∈ A ∩ B) ≡ ¬(x∈A ∧ x∈B).

Step 3: Apply De Morgan's law from propositional logic: ¬(x∈A ∧ x∈B) ≡ (x∉A) ∨ (x∉B).

Step 4: Translate back: (x∉A) ∨ (x∉B) ≡ x ∈ A̅ ∪ B̅.

Step 5: Therefore A∩B = A̅ ∪ B̅. ✓

Rosen 2.2 #12 Set Identity▾

Prove (A∩B)∪C = C ∪ (A∩B) using commutativity.

Solution

Step 1: The union operation ∪ is commutative: for any sets X,Y, X∪Y = Y∪X.

Step 2: Let X = (A∩B) and Y = C. Then (A∩B)∪C = C∪(A∩B) by commutativity.

Step 3: No further simplification needed — the identity follows directly from the definition of set union.

M4CS 4.1 #8 Set Difference▾

Prove A\(B\C) = (A\B)∪(A∩C).

Solution

Step 1: Translate LHS: x ∈ A\(B\C) ≡ x∈A ∧ ¬(x∈B ∧ x∉C).

Step 2: Apply De Morgan: x∈A ∧ (x∉B ∨ x∈C).

Step 3: Distribute: (x∈A ∧ x∉B) ∨ (x∈A ∧ x∈C).

Step 4: Translate back: (x∈A\B) ∨ (x∈A∩C) ≡ x ∈ (A\B) ∪ (A∩C).

Step 5: Therefore A\(B\C) = (A\B) ∪ (A∩C). ✓

Key Insight: Converting set expressions to propositional logic and back is a systematic method for proving set identities.

Solution

Step 2 (Inductive hypothesis): Assume ∑_{i=1}^{n} i² = n(n+1)(2n+1)/6 for some n ≥ 1.

Step 3 (Inductive step): Show for n+1: ∑_{i=1}^{n+1} i² = ∑_{i=1}^{n} i² + (n+1)².

Step 4: Substitute the hypothesis: = n(n+1)(2n+1)/6 + (n+1)².

M4CS 5.1 — Divisibility by 9▾

Solution

Step 2 (Inductive hypothesis): Assume 4ⁿ + 15n − 1 = 9k for some integer k.

Step 3 (Inductive step): Compute for n+1: 4^{n+1} + 15(n+1) − 1 = 4·4ⁿ + 15n + 14.

M4CS 5.1 — Chocolate Bar▾

An m×n bar needs mn−1 splits regardless of sequence.

Strong Induction

Step 1 (Proposition P(s)): A chocolate bar with s squares requires exactly s−1 splits to separate into individual squares.

Step 2 (Base s=1): Already a single square, 0 splits needed. 0 = 1−1. ✓

Step 3 (Inductive hypothesis): Assume P(k) holds for all k ≤ s (strong induction).

Step 4 (Inductive step): For a bar of s+1 squares, any first split divides it into two pieces of sizes a and b where a + b = s+1.

Step 5: By the strong IH, the a-sized piece needs a−1 splits and the b-sized piece needs b−1 splits.

Step 6: Total splits = 1 (first split) + (a−1) + (b−1) = a + b − 1 = (s+1) − 1 = s. ✓

M4CS 5.1 — Stacking Game▾

Split pile s into a,b (a+b=s), score a·b. Total always n(n-1)/2.

Step 1 (Base n=1): One square, no split possible. Score = 0 = 1·0/2. ✓

Step 2 (Inductive hypothesis): Assume for piles smaller than n, total score = k(k−1)/2.

Step 3 (Inductive step): For a pile of size n, first split makes piles of sizes a and b where a+b=n, giving score a·b.

Step 4: By IH, the a-pile contributes a(a−1)/2 and the b-pile contributes b(b−1)/2.

Step 3 (Inductive step): ∑_{k=1}^{n+1} 1/k² = ∑_{k=1}^{n} 1/k² + 1/(n+1)² < 2 − 1/n + 1/(n+1)².

Why the 14/15 swapped state is unreachable.

Invariant

MIT R03 — Temple of Forever▾

Solutions

Step 1 (1): f∼g means lim f/g = 1, so lim (f−g) = 0. Then lim 2^f/2^g = lim 2^{f-g} = 2^0 = 1. So 2^f ∼ 2^g. ✓

Step 2 (2): lim f²/g² = lim (f/g)² = 1² = 1. So f² ∼ g². ✓

Step 3 (3): Let f(n) = n, g(n) = n+1. Then f/g = n/(n+1) → 1, so f∼g. But 2^{n+1}/2ⁿ = 2 → 2, which is not 1. So 2^f ≁ 2^g.

Step 4 (4): Reflexive: f/f → 1. Symmetric: if f/g → 1 then g/f → 1/1 = 1. Transitive: if f/g → 1 and g/h → 1, then f/h = (f/g)(g/h) → 1·1 = 1. ✓

Step 5 (5): Θ also satisfies reflexivity (f = Θ(f)), symmetry (if f=Θ(g) then g=Θ(f)), and transitivity (if f=Θ(g) and g=Θ(h) then f=Θ(h)). Though Θ's equivalence classes are coarser than ∼'s.

Step 1: Assuming f∼g implies 2^f ∼ 2^g. This is false because exponentiation amplifies additive differences.

Step 2: Counterexample: n ∼ n+1 (ratio → 1), but 2ⁿ⁺¹/2ⁿ = 2, so 2ⁿ ≁ 2^{n+1}.

Key Insight: Asymptotic equivalence is preserved under multiplication, addition, and roots, but NOT under exponentiation.

MIT A08 Q2 — Bogus Asymptotic Induction▾

Step 2: Inductive hypothesis: T(n/2) ≤ c·(n/2).

Step 3: Then T(n) = 4T(n/2) + n ≤ 4·c·(n/2) + n = 2cn + n = (2c+1)n.

Step 2: Simply assuming T(n/2) = O(n/2) with the same c fails because the recurrence multiplies c by 4 and divides by 2, producing net factor 2, which can't be absorbed.

Step 3 — (c) 7T(n/20) + 2T(n/8) + n: Two distinct subproblem sizes: Master Theorem does not apply. Use Akra-Bazzi with a₁=7, b₁=1/20, a₂=2, b₂=1/8, g(n)=n. Solve 7(1/20)^p + 2(1/8)^p = 1. p ≈ 0.16. Since p < 1, the integral ∫₁ⁿ u/u^{p+1} du diverges, giving T(n) = Θ(n).

Step 4 — (d) 2T(n/4 + 1) + √n: a=2, b=4, d=0.5 (since √n = n⁰·⁵). The +1 shift is negligible. Compute log₄ 2 = 0.5. Since a = bᵈ, Case 2 applies: T(n) = Θ(√n log n).

Step 5 — (e) 3T(n/9 + n^{1/9}) + 1: a=3, b=9, d=0 (since f(n)=1 = n⁰). The n^{1/9} perturbation is negligible. Compute log₉ 3 = 0.5. Since 0.5 > 0, Case 1 applies: T(n) = Θ(n^{log₉ 3}) = Θ(n⁰·⁵) = Θ(√n).

Key Insight: Small additive perturbations (±1, +1/n, +n^{1/9}) do not affect the asymptotic classification. Always check if they are dominated by the main term before accounting for them.

MIT A08 Q3 — Linear Recurrence▾

A₀ = 2, A_{n+1} = A_n/2 + 1/A_n. Prove A_n ≤ √2 + 1/2ⁿ for all n≥0.

Solution

Step 1 (Base case n=0): A₀ = 2. The claim is 2 ≤ √2 + 1/2⁰ = √2 + 1 ≈ 2.414. Since 2 ≤ 2.414, the base case holds.

Step 2 (Inductive hypothesis): Assume A_n ≤ √2 + 1/2ⁿ for some n ≥ 0.

Step 3 (Inductive step): Compute A_{n+1} = A_n/2 + 1/A_n. Using the hypothesis, A_n/2 ≤ (√2 + 1/2ⁿ)/2 = √2/2 + 1/2^{n+1}.

Step 4 (Bound 1/A_n): Since A_n ≥ 0 (non-negative by definition), we need a lower bound on A_n to bound 1/A_n from above. Note A_n approaches √2 from above, so 1/A_n ≤ 1/√2 = √2/2.

Step 5 (Combine bounds): A_{n+1} ≤ √2/2 + 1/2^{n+1} + √2/2 = √2 + 1/2^{n+1}. This completes the induction.

Key Insight: The recurrence converges to √2. The induction uses the identity 1/√2 = √2/2 to simplify the sum. Each iteration halves the error term.

MIT A08 Q4 — Solving Linear Recurrences▾

Find closed-form solutions:

(a) x_n = 4x_{n-1} − x_{n-2} − 6x_{n-3}, x₀=3, x₁=4, x₂=14.

(b) x_n = −x_{n-1} + 2x_{n-2} + n, x₀=5, x₁=−4/9.

Solution

Step 1 — (a) Characteristic equation: The homogeneous recurrence x_n = 4x_{n-1} − x_{n-2} − 6x_{n-3} gives characteristic r³ − 4r² + r + 6 = 0.

Step 2 — (a) Factor: Factor the cubic: (r+1)(r-2)(r-3) = 0. Roots are r = −1, 2, 3 (all distinct).

Step 3 — (a) General form: With three distinct roots, x_n = A(−1)ⁿ + B·2ⁿ + C·3ⁿ.

Step 4 — (a) Solve for constants: Use initial conditions: n=0: A + B + C = 3. n=1: −A + 2B + 3C = 4. n=2: A + 4B + 9C = 14. Solving gives A=1, B=1, C=1. Therefore x_n = (−1)ⁿ + 2ⁿ + 3ⁿ.

Step 5 — (b) Homogeneous solution: The homogeneous part r² + r − 2 = 0 factors as (r-1)(r+2) = 0, with roots r=1, r=−2.

Step 6 — (b) Particular solution attempt: Since the non-homogeneous term is n (degree 1) and r=1 is a root, try x_n = n(pn+q) = pn²+qn. Substitute into the recurrence to solve for p, q.

Step 7 — (b) Find p, q: Substituting x_n = pn²+qn: pn²+qn = −(p(n-1)²+q(n-1)) + 2(p(n-2)²+q(n-2)) + n. Collecting coefficients gives p = 1/2, q = −5/6.

Step 8 — (b) Complete solution: x_n = A·1ⁿ + B·(−2)ⁿ + n²/2 − 5n/6 + ... (the particular solution may need a constant term adjustment since r=1 is a root).

Key Insight: When the non-homogeneous term overlaps with a characteristic root, multiply the particular solution guess by n (or n^m for multiplicity m) to avoid duplication.

MIT R14 — TriMergeSort▾

TriMergeSort splits into 3 sublists of n/3, sorts recursively, merges with 2 comparisons per 3 items.

Find T(n) recurrence and Θ bound.

Solution

Step 1: Write the recurrence. Three subproblems each of size n/3, and merging three sorted lists of total size n requires 2n−3 comparisons in the worst case. So T(n) = 3T(n/3) + 2n.

Step 2: Apply Master Theorem: a=3, b=3, d=1 (since 2n = Θ(n¹)). Compute log_b a = log₃ 3 = 1.

Step 3: Compare: a = bᵈ (3 = 3¹), so Master Theorem Case 2 applies: T(n) = Θ(n log n).

Key Insight: Splitting into 3 subproblems instead of 2 does not change the Θ class. TriMergeSort is still Θ(n log n), same as ordinary MergeSort.

Solution — Akra-Bazzi Verification

Step 1: Apply Akra-Bazzi to verify: a=3, b=1/3, g(n)=2n. Solve 3·(1/3)^p = 1.

Step 2: Simplify: 3^{1-p} = 1 → 1-p = 0 → p = 1.

Step 3: With p=1, evaluate the Akra-Bazzi formula: T(n) = Θ(n(1 + ∫₁ⁿ g(u)/u² du)) = Θ(n(1 + ∫₁ⁿ 2u/u² du)) = Θ(n(1 + 2∫₁ⁿ 1/u du)).

Step 4: Evaluate the integral: ∫₁ⁿ 1/u du = ln n. Therefore T(n) = Θ(n(1 + 2 ln n)) = Θ(n log n).

Key Insight: Akra-Bazzi confirms the Master Theorem result. Both methods converge to the same asymptotic bound.

MIT R14 — Linear Recurrences▾

Find closed forms:

(1) T₀=0, T₁=1, T_n = T_{n-1} + T_{n-2} + 1.

(2) S₀=0, S₁=1, S_n = 6S_{n-1} − 9S_{n-2}.

Solution

Step 1 — (1) Homogeneous part: The homogeneous recurrence T_n = T_{n-1} + T_{n-2} is the Fibonacci recurrence with characteristic r² − r − 1 = 0. Roots are φ = (1+√5)/2 and −1/φ = (1−√5)/2.

Step 2 — (1) Particular solution: Try constant T_n = c. Substituting: c = c + c + 1 → c = −1. So the particular solution is T_n = −1.

Step 3 — (1) General form: T_n = A·φⁿ + B·(−1/φ)ⁿ − 1. Using T₀ = 0: A + B − 1 = 0 → A + B = 1. Using T₁ = 1: Aφ − B/φ − 1 = 1 → Aφ − B/φ = 2.

Step 4 — (1) Solve and simplify: Solving the system and recognizing the Fibonacci pattern yields T_n = F_{n+2} − 1 where F₁=1, F₂=1. Verify: n=0 → F₂−1 = 1−1 = 0 ✓, n=1 → F₃−1 = 2−1 = 1 ✓.

Step 5 — (2) Characteristic equation: r² − 6r + 9 = 0 factors as (r−3)² = 0, giving a repeated root r = 3 with multiplicity 2.

Step 7 — (2) Final closed form: S_n = n·3^{n-1}.

Key Insight: Repeated roots require the polynomial prefactor (A + Bn) instead of separate exponential terms. This is analogous to repeated roots in differential equations.

Rosen 8.1 #1 — Bit Strings▾

Problem: Find a recurrence for the number of bit strings of length n that do NOT have two consecutive 0s.

Solution

Step 1: Let a_n be the number of valid bit strings of length n (no consecutive 0s). Partition the set by the last digit.

Step 2 — Strings ending in 1: If the last digit is 1, the first n−1 bits can be any valid string of length n−1. This contributes a_{n-1} strings.

Step 3 — Strings ending in 0: If the last digit is 0, the preceding digit must be 1 (otherwise we'd have consecutive 0s). So the string ends in "10", and the first n−2 bits can be any valid string of length n−2. This contributes a_{n-2} strings.

Step 4 — Recurrence: a_n = a_{n-1} + a_{n-2}. Base cases: a₁ = 2 (strings "0" and "1"), a₂ = 3 (strings "01", "10", "11"; "00" is invalid).

Key Insight: Partition by last digit. The constraint "no 00" forces a trailing 0 to be preceded by 1, which reduces the problem to counting valid prefixes of length n−2.

Rosen 8.2 #4 — Linear Recurrence▾

Problem: Solve recurrence: a_n = 7a_{n-1} − 10a_{n-2} with a_0 = 2, a_1 = 1.

Solution

Step 1: Write the characteristic equation: r² − 7r + 10 = 0.

Step 2: Factor: (r−2)(r−5) = 0. Roots are r = 2 and r = 5 (distinct).

Step 3: General solution with distinct roots: a_n = A·2ⁿ + B·5ⁿ.

Step 4: Apply initial condition a₀ = 2: A·2⁰ + B·5⁰ = A + B = 2.

Step 5: Apply initial condition a₁ = 1: A·2¹ + B·5¹ = 2A + 5B = 1.

Step 6: Solve the system: From A + B = 2, B = 2 − A. Substitute: 2A + 5(2−A) = 1 → 2A + 10 − 5A = 1 → −3A = −9 → A = 3, B = −1.

Step 7: Final closed form: a_n = 3·2ⁿ − 5ⁿ.

Common Mistake: Forgetting to use both initial conditions to determine the constants A and B. One condition alone leaves a free parameter.

Rosen 8.2 #12 — Repeated Root▾

Problem: Solve b_n = 8b_{n-1} − 16b_{n-2} with b_0 = 1, b_1 = 4.

Solution

Step 1: Write the characteristic equation: r² − 8r + 16 = 0.

Step 2: Factor: (r−4)² = 0. This gives a single root r = 4 with multiplicity 2.

Step 3: For a repeated root of multiplicity 2, use the form b_n = (A + Bn)·4ⁿ.

Step 4: Apply b₀ = 1: (A + B·0)·4⁰ = A = 1.

Step 5: Apply b₁ = 4: (1 + B·1)·4¹ = 4 → (1 + B)·4 = 4 → 1 + B = 1 → B = 0.

Step 6: Final closed form: b_n = 4ⁿ.

Common Mistake: Using A·4ⁿ + B·4ⁿ (which collapses to (A+B)·4ⁿ, a single exponential term) instead of (A + Bn)·4ⁿ for repeated roots. The factor n provides the second independent solution.

Rosen 8.3 #1 — Binary Search Comparisons▾

Problem: Binary search on n items. Worst-case comparisons f(n) = f(⌊n/2⌋) + 2 for n ≥ 2, f(1) = 2. Find big-O bound.

Solution

Step 1: Identify Master Theorem parameters. The recurrence is f(n) = f(n/2) + 2, so a=1 (one subproblem), b=2 (halving), d=0 (since f(n) = 2 = Θ(n⁰)).

Step 2: Compute log_b a = log₂ 1 = 0. Compare with d = 0.

Step 3: Since a = bᵈ (1 = 2⁰), Master Theorem Case 2 applies: f(n) = Θ(nᵈ log n) = Θ(log n).

Step 4: More concretely, the recursion depth is log₂ n and each level does 2 comparisons, so f(n) = 2 log₂ n + 2 = Θ(log n).

Common Mistake: Binary search does 2 comparisons per level (one equality check and one less-than check), not 1. However, the constant factor does not affect the asymptotic Θ(log n) bound.

Rosen 8.3 #13 — Master Theorem Cases▾

Problem: For each recurrence, find Θ bound: (a) T(n) = 4T(n/2) + n (b) T(n) = 4T(n/2) + n² (c) T(n) = 4T(n/2) + n³.

Solution

Step 1 — Common parameters: All three have a=4, b=2, so log_b a = log₂ 4 = 2. This is the threshold exponent.

Step 2 — (a) T(n) = 4T(n/2) + n: f(n) = n = n¹. Compare exponent 1 with threshold 2: 1 < 2, so Case 1 applies. T(n) = Θ(n^{log_b a}) = Θ(n²). The recursion dominates.

Step 3 — (b) T(n) = 4T(n/2) + n²: f(n) = n² matches the threshold n² exactly. Case 2 applies: T(n) = Θ(n² log n).

Step 4 — (c) T(n) = 4T(n/2) + n³: f(n) = n³, exponent 3 > threshold 2. Case 3 potentially applies, but verify regularity: 4·(n/2)³ = 4·n³/8 = n³/2 ≤ c·n³ for c = 1/2 < 1. The regularity condition holds, so T(n) = Θ(n³). The non-recursive work dominates.

Common Mistake: Confusing which case applies based on exponent comparison. Remember: the threshold exponent is log_b a. Compare the exponent of f(n) against it, not against d alone.

8. Meta-Logic — Fallacies, Consistency, Completeness, Gödel Set 4

Set 4 Logical Fallacies▾
Fallacy: An error in reasoning that results in an invalid argument. Fallacies resemble valid rules of inference but are based on contingencies rather than tautologies. (Rosen §1.6.6)
Set 4 Rosen §1.6.6 — Affirming the Conclusion▾
Fallacy of Affirming the Conclusion: The proposition ((p→q) ∧ q) → p is not a tautology — it is false when p is false and q is true. Despite this, many incorrect arguments treat it as valid.

Example (Rosen): "If you do every problem in this book, then you will learn discrete mathematics. You learned discrete mathematics. Therefore, you did every problem in this book."

Why it fails

Step 1: Let p = "You did every problem," q = "You learned DM." The argument is: p→q, q, therefore p.

Step 2: You could learn DM by reading, attending lectures, or doing some (not all) problems. The premise q (you learned DM) could be true while p (you did every problem) is false.

Step 3: This is exactly the truth table row where p is false and q is true: p→q is true (F→T = T), q is true, but the conclusion p is false. So the inference is invalid.

Key Insight: Affirming the conclusion confuses a sufficient condition (doing all problems is enough to learn DM) with a necessary one (it's not the only way).

How to Spot

If an argument says "If P then Q, Q is true, therefore P is true," it's committing this fallacy. The correct inference is modus ponens: P→Q, P, therefore Q.

Set 4 Rosen §1.6.6 — Denying the Hypothesis▾
Fallacy of Denying the Hypothesis: The proposition ((p→q) ∧ ¬p) → ¬q is not a tautology — it is false when p is false and q is true.

Example (Rosen): "If you do every problem in this book, then you will learn discrete mathematics. You did not do every problem. Therefore, you did not learn discrete mathematics."

Why it fails

Step 1: Let p = "You did every problem," q = "You learned DM." The argument is: p→q, ¬p, therefore ¬q.

Step 2: Even without doing every problem, you could still learn DM through other means. So ¬p (didn't do every problem) can be true while q (learned DM) is also true, making ¬q false.

Step 3: This is the truth table row p=F, q=T: p→q is true, ¬p is true, but the conclusion ¬q is false.

Key Insight: Denying the hypothesis confuses a sufficient condition with a necessary condition. The correct inference is modus tollens: P→Q, ¬Q, therefore ¬P.

Set 4 Rosen §1.8 — Fallacious "Proof" Examples▾

Problem (Rosen Ex 17): Find the error in this "proof": "If n² is positive, then n is positive."

Solution

Step 1: The argument claims: "If n is positive, then n² is positive" (∀n, P(n)→Q(n)). We have Q(n) (n² is positive), so conclude P(n) (n is positive).

Step 2: This is the fallacy of affirming the conclusion. Counterexample: n = -1 gives n² = 1 (positive) but n is negative.

Key Insight: Always test a proof with edge cases (negative numbers, zero, etc.) before accepting it.

Problem (Rosen Ex 18): Find the error: "If n is not positive, then n² is not positive."

Solution

Step 1: The argument claims: from ¬P(n) (n not positive) and ∀n (P(n)→Q(n)), conclude ¬Q(n) (n² not positive).

Step 2: This is the fallacy of denying the hypothesis. Counterexample: n = -1 is not positive, but n² = 1 is positive.

Set 4 Rosen §1.8 — Begging the Question (Circular Reasoning)▾
Begging the Question: A fallacy where one or more steps of a proof are based on the truth of the statement being proved — the conclusion is assumed in the premises (circular reasoning).

Example (Rosen Ex 19): "Suppose n² is even. Then n² = 2k for some integer k. Let n = 2l for some integer l. This shows n is even."

Why it fails

Step 1: The statement "let n = 2l for some integer l" is exactly what we're trying to prove — that n is even.

Step 2: No justification is given for why n can be written as 2l. The proof assumes its own conclusion.

Key Insight: The statement itself (if n² even then n even) is actually true — but the proof is invalid because it uses circular reasoning. A correct proof requires the contrapositive or prime factorization.

Set 4 Web / Extra — Straw Man Fallacy▾
Straw Man: Misrepresenting someone's argument to make it easier to attack. Instead of addressing the actual argument, you attack a distorted or exaggerated version of it.

Example (DM context): Person A: "I think we need to prove the base case before the inductive step." Person B: "So you're saying we should memorize base cases instead of understanding the inductive hypothesis? That's ridiculous."

Why it fails

Step 1: Person A's actual position: the base case is a necessary component of an inductive proof.

Step 2: Person B misrepresents this as "memorize base cases instead of understanding induction" — which is not what Person A said at all.

Step 3: Person B then attacks this made-up position and claims victory. The original argument was never addressed.

Key Insight: In math proofs, straw man arguments waste time by disproving claims nobody made. Always restate your opponent's position in your own words to confirm understanding before rebutting.

Set 4 Web / Extra — False Dilemma (False Dichotomy)▾
False Dilemma: Presenting only two options when more exist. This forces a choice between extremes when intermediate positions are available.

Example (DM context): "Either P=NP or computer science is useless. P=NP hasn't been proven yet. Therefore, computer science is useless."

Why it fails

Step 1: The argument presents only two possibilities: P=NP or CS is useless. This ignores the third option: P≠NP but CS is still useful.

Step 2: Computer science has produced countless important results (cryptography, algorithms, programming languages, etc.) regardless of the P vs NP question.

Step 3: The false dilemma improperly restricts the universe of possibilities.

Key Insight: Proof by cases is only valid when the cases are exhaustive. A false dilemma smuggles in non-exhaustive cases. Always check: "Are there other possibilities not considered?"

DM Connection

This fallacy is especially dangerous in proofs by cases — if your cases don't cover all possibilities, the proof is invalid. For example, proving something for "n even" and "n odd" covers all integers. But proving it for "n positive" and "n negative" misses n=0.

Set 4 Web / Extra — Slippery Slope▾
Slippery Slope: Arguing that a relatively small first step inevitably leads to a chain of events culminating in some (usually bad) effect, without providing evidence for the chain of causation.

Example (DM context): "If we allow the inductive hypothesis to assume P(k) for k < n, then next we'll be assuming P(k) for all k without proof, and then we won't need base cases at all, and mathematics will collapse!"

Why it fails

Step 1: The argument claims that assuming P(k) for k < n (strong induction) leads to "mathematics collapsing" — but provides no evidence for this chain.

Step 2: Strong induction is a legitimate, well-understood proof technique with a solid logical foundation. It does not lead to abandoning base cases.

Step 3: Each step in the chain requires its own justification. A valid inductive proof explicitly shows each implication P(k) → P(k+1); a slippery slope skips these justifications.

Key Insight: The slippery slope is a "false implication" — it asserts P→Q→R without proving any of the intermediate arrows. Compare with mathematical induction, where each arrow (inductive step) is explicitly proved.

Set 4 Web / Extra — Appeal to Ignorance (Argumentum ad Ignorantiam)▾
Appeal to Ignorance: Claiming something is true because it hasn't been proven false (or vice versa). The burden of proof is shifted incorrectly.

Example (DM context): "No one has found a counterexample to Goldbach's conjecture after centuries of searching. Therefore, Goldbach's conjecture is true."

Why it fails

Step 1: The absence of a counterexample is not the same as a proof. There might be a counterexample that nobody has found yet.

Step 2: In discrete mathematics, many conjectures seemed true for large ranges until counterexamples were found (e.g., the Borwein integral pattern that breaks at n=15).

Step 3: The correct logical form: ¬∃x ¬P(x) does NOT imply ∀x P(x) unless the domain is finite and we've checked every element.

Key Insight: This is connected to the limits of induction in math — checking finitely many cases never proves a universal statement over an infinite domain. That's why we need mathematical induction, not empirical checking.

Common Mistake

Mistake: Thinking "no counterexample found" equals "the statement is true."

Why it fails: In mathematics, conjectures require proof, not evidence. A statement proved true for n up to 10⁶ might still fail at 10⁶+1. This is why induction proofs must prove the inductive step for all n — not just test it on finitely many values.

Set 4 Web / Extra — Hasty Generalization▾
Hasty Generalization: Drawing a general conclusion from insufficient or unrepresentative evidence. The sample size is too small to support the conclusion.

Example (DM context): "I tested the formula n² + n + 41 for n = 0, 1, 2, 3, 4, and it gave prime numbers every time. Therefore, n² + n + 41 produces primes for all n ≥ 0."

Why it fails

Step 1: Testing 5 values is insufficient to prove a universal claim over all natural numbers (an infinite set).

Step 2: Indeed, n² + n + 41 = 41² + 41 + 41 = 41·43 = 1763 (composite) when n = 41. The pattern fails.

Step 3: This is a classic example in discrete mathematics showing why proof by example doesn't work for universal statements.

Key Insight: Hasty generalization is the informal counterpart to the mathematical error of "proof by insufficient base cases." In induction, you need the base case and the inductive step — many examples alone never constitute a proof.

Common Mistake

Mistake: In induction proofs, forgetting to verify the inductive step and thinking "I checked the first 10 cases" is sufficient. This is hasty generalization.

Why it fails: P(0) through P(9) being true doesn't mean P(10) is true unless you've proved P(k) → P(k+1). The inductive step is what bridges the gap from finite checking to infinite coverage.

Set 4 Web / Extra — Red Herring▾
Red Herring: Introducing an irrelevant topic to divert attention from the original issue. The argument changes the subject to something easier to address.

Example (DM context): "My proof of the recurrence has a small gap, but the algorithm runs in linear time anyway, so it doesn't matter."

Why it fails

Step 1: The claim about the algorithm's runtime is a separate issue from whether the recurrence proof is correct.

Step 2: Even if the algorithm is fast in practice, the proof still needs to be logically valid. A "small gap" could hide a fundamental error.

Step 3: The runtime claim (even if true) doesn't fix the gap in the proof. Two correct claims don't make an incorrect argument correct.

Key Insight: In mathematical proofs, every step must be justified. Introducing irrelevant true facts doesn't patch gaps in reasoning. Focus on the logical structure, not tangential truths.

Set 4 M4CS 3.33 — The Converse Fallacy▾

Problem (M4CS 3.33): The formula (P→Q) ∨ (Q→P) is valid (always true). This suggests a "proof method": to prove an implication is true, just prove its converse is false. But this leads to paradox. Explain the flaw.

Solution

Step 1: The validity of (P→Q) ∨ (Q→P) means at least one is true — but it doesn't tell us which one.

Step 2: If we know Q→P is false, then P→Q must be true (by the validity). So knowing the converse is false does let us conclude the original is true — if we know the converse is false.

Step 3: The fallacy is in thinking we can determine the converse is false without determining the truth of the original. In the calculus example: "If continuous then differentiable" is false, so its converse "If differentiable then continuous" should be true. But this works because we independently know the original is false.

Step 4: The second example fails: "If differentiable then not continuous" is false. The method would claim its converse "If not continuous then differentiable" is true — but that converse is also false. Why? Because we were wrong about the original being false — we assumed it was false without properly verifying.

Key Insight: The "method" only works when we have an independent way to determine the truth of the original implication. The validity of (P→Q) ∨ (Q→P) is about existence, not about which one is true.

Common Mistake

Mistake: Treating (P→Q) ∨ (Q→P) as if it provides a constructive way to determine truth.

Why it fails: A tautology tells you that at least one disjunct is true, not which one. You still need separate reasoning to determine the truth of each component.

Set 4 Medium — Mathematical Fallacies (Rosen §1.8 + Extra)▾

Find the error in each mathematical "proof":

(a) All horses are the same color (Rosen §1.8 — Induction Fallacy): We prove by induction that for all n ≥ 1, any group of n horses are the same color. Base case n=1: one horse trivially has the same color as itself. Inductive step: Assume any n horses are the same color. Given n+1 horses, remove one to get n horses — by IH they're the same color. Return the removed horse and remove a different one — again by IH these n are the same color. Since the two groups overlap, all n+1 horses are the same color.

(b) 1 = 2 (Algebraic Fallacy): Let a = b. Then a² = ab. Subtract b²: a² − b² = ab − b². Factor: (a−b)(a+b) = b(a−b). Divide by (a−b): a+b = b. Since a = b: b+b = b, so 2b = b, so 2 = 1.

(c) Every integer can be described in fourteen words (Berry Paradox): Consider the set of integers that cannot be described in fourteen words. This set is non-empty (there are only finitely many fourteen-word descriptions but infinitely many integers). Take the smallest such integer: "The smallest positive integer that cannot be described in fourteen words." That description has fourteen words — contradiction! Therefore, every integer can be described in fourteen words.

(d) 1 = −1 (Square Root Fallacy): 1 = √1 = √((−1)(−1)) = √(−1) · √(−1) = i · i = −1.

(e) All triangles are isosceles (Geometric Fallacy): Given any triangle ABC, draw the perpendicular bisector of BC and the angle bisector at A. Let their intersection be O. Draw perpendiculars from O to AB and AC. Through a series of angle-side-angle congruences, we "prove" that AB = AC — every triangle is isosceles.

Solutions — Where Each Proof Goes Wrong

(a) All horses: The inductive step fails at n=1→2. When n+1=2, removing one horse gives a group of 1 (trivially same color). But the two groups {horse 1} and {horse 2} are disjoint — they don't overlap. The argument that "the two groups overlap" only works when n ≥ 2, so the base case needs to be n=2 (but that's false!). Fallacy type: Induction with insufficient base case coverage.

(b) 1 = 2: Division by (a−b) is division by zero (since a=b). Division by zero is undefined — the step is invalid. Fallacy type: Invalid algebraic operation (division by zero).

(c) Berry Paradox: The phrase "described in fourteen words" is ambiguous — it's a meta-linguistic description, not a formal mathematical one. The paradox reveals the problem of self-reference in natural language, not a genuine contradiction in mathematics. Unlike Gödel's formal self-reference, this informal paradox doesn't have a well-defined notion of "description." Fallacy type: Self-referential paradox (informal language).

(d) 1 = −1: The identity √(ab) = √a · √b only holds for non-negative real numbers. When a = b = −1, both are negative, so the splitting is invalid. Fallacy type: Invalid application of a property outside its domain.

(e) All triangles isosceles: The intersection point O lies outside the triangle for non-isosceles triangles — the perpendiculars from O to AB and AC don't fall on the segments but on their extensions. The side-length equalities derived are between external segments, not the triangle's actual sides. Fallacy type: Hidden assumption about point location (incomplete case analysis).

Key Insight: Mathematical fallacies teach us to check: (1) Are we dividing by zero? (2) Does the base case actually support the induction? (3) Is the property valid for all elements in the domain? (4) Are there hidden geometric assumptions? (5) Is the self-reference well-defined?

Set 4 Medium — Set 13: Parity & Number Theory Proof Spotting (Rosen §1.7–1.8 + M4CS Bogus Proofs)▾

Find the error in each "proof" — the statement may be true, but the proof is invalid:

(a) Missing case (Rosen §1.7 #3 variant): "Prove: For all integers n, n² − n is even."
Proof: "Case 1: n is even, n = 2k. Then n² − n = 4k² − 2k = 2(2k² − k), which is even. Therefore n² − n is even for all n."

(b) Extraneous solutions (Rosen §1.8 #36): "Solve √(2x − 1) = x − 2."
Proof: "Square both sides: 2x − 1 = x² − 4x + 4. Rearr: 0 = x² − 6x + 5. Factor: 0 = (x − 1)(x − 5). So x = 1 or x = 5."

(c) Converse error (Rosen §1.7 #20 variant): "Prove: If 3n + 2 is even, then n is even."
Proof: "Assume n is even: n = 2k. Then 3n + 2 = 6k + 2 = 2(3k + 1), which is even. ✓ Therefore if 3n + 2 is even, n is even."

(d) Confusing even with divisible by 4: "Prove: If n² is divisible by 4, then n is divisible by 4."
Proof: "Assume n² = 4k for some integer k. Then n = √(4k) = 2√k. Since n is an integer, √k must be an integer, so k = m². Then n² = 4m², so n = 2m. Therefore n is divisible by 4."

(e) Sign reversal (M4CS §1 Bogus Proof): "Prove: 1/8 > 1/4."
Proof: "3 > 2. Multiply both sides by log(1/2): 3·log(1/2) > 2·log(1/2). Using log properties: log((1/2)³) > log((1/2)²). Since log is increasing: (1/2)³ > (1/2)². Computing: 1/8 > 1/4. ✓"

(f) Divisibility non-sequitur (Rosen §1.8 #18 variant): "Prove: If m and n are integers and mn is even, then m and n are both even."
Proof: "Assume mn is even. Then mn = 2k. Since 2 is prime, either m or n must contain the factor 2. But if only one is even, the product would have only one factor of 2. Wait... Actually, since the product is even, both m and n must be even."

Solutions — Where Each Proof Goes Wrong

(a) Missing case: The proof only covers the "n even" case. It never considers n odd. (The claim is true, but the proof is incomplete. When n = 2k+1: n² − n = (4k²+4k+1) − (2k+1) = 4k²+2k = 2(2k²+k), also even — but the proof didn't show this.) Error type: Incomplete case analysis.

(b) Extraneous solutions: Squaring both sides can introduce false solutions. We must check each candidate: x = 5 → √(2·5−1) = √9 = 3 and 5−2 = 3 ✓. x = 1 → √(2·1−1) = √1 = 1 but 1−2 = −1 ✗. Only x = 5 is valid. Error type: Missing verification step (loss of logical equivalence).

(c) Converse error: The proof shows "n even ⇒ 3n + 2 even" — which is the CONVERSE of the required statement. The required statement is "3n + 2 even ⇒ n even." We proved the wrong direction. (Affirming the conclusion.) A correct proof uses the contrapositive: if n is odd, then 3n + 2 = 3(2k+1) + 2 = 6k + 5 = 2(3k+2) + 1 is odd. Error type: Proving the converse instead of the original (affirming the conclusion).

(d) Confusing even with divisible by 4: n = 2m means n is EVEN, not divisible by 4! Counterexample: n = 2 gives n² = 4 (divisible by 4) but n = 2 is not divisible by 4. The proof correctly shows n is even (n = 2m) but incorrectly concludes divisible by 4. Error type: Non-sequitur — the conclusion doesn't follow from the derived fact.

(e) Sign reversal: log(1/2) = log(2⁻¹) = −log(2) which is NEGATIVE. Multiplying an inequality by a negative number reverses the direction: 3 > 2 should become 3·log(1/2) < 2·log(1/2). The entire chain is invalid. Error type: Multiplying inequality by a negative number without flipping the sign.

(f) Divisibility non-sequitur: The conclusion that "both m and n are even" is too strong. Counterexample: m = 3 (odd), n = 2 (even) gives mn = 6 (even). The correct conclusion is: if mn is even, then at least one of m, n is even (possibly both). The proof's "both must be even" doesn't follow. Error type: Overgeneralization — the conclusion is stronger than what the premises justify.

Key Insight: When spotting errors in proofs, ask: (1) Are all cases covered? (2) Are there extraneous solutions? (3) Is the direction of implication correct? (4) Does the conclusion actually follow? (5) Are operations valid (especially inequality sign flips)? (6) Is the conclusion too strong?

Set 4 Consistency▾
Set 4 Definition & Intuition▾
Consistency (Propositional Logic — M4CS 3.16): A set of propositional formulas {P₁, ..., Pₖ} is consistent iff there exists a truth assignment (environment) in which all of them are simultaneously true.
Consistency (Axiom Systems — LS01): A set of axioms is consistent when you cannot prove that False is True (i.e., no contradiction can be derived).
Syntactic vs Semantic Consistency: Syntactic consistency means no proof of a contradiction exists. Semantic consistency means there is a model (an interpretation) where all axioms hold. For propositional logic, these coincide (soundness and completeness theorems).
Principle of Explosion (Ex Falso Quodlibet): From a contradiction, any statement follows. Formally: (P ∧ ¬P) → Q is a tautology for any Q. This is why consistency is essential — a single contradiction makes the entire system trivial (everything becomes provable).
Set 4 M4CS 3.16 — Consistency & Satisfiability▾

Problem (M4CS 3.16): A set of propositional formulas {P₁, ..., Pₖ} is consistent iff there is an environment in which they are all true. Write a formula S such that the set is not consistent iff S is valid.

Solution

Step 1: The set is consistent iff there exists an assignment where P₁ ∧ P₂ ∧ ... ∧ Pₖ is true. In other words, the conjunction is satisfiable.

Step 2: The set is inconsistent (not consistent) iff the conjunction P₁ ∧ P₂ ∧ ... ∧ Pₖ is unsatisfiable.

Step 3: A formula is unsatisfiable iff its negation is valid. So the set is inconsistent iff ¬(P₁ ∧ P₂ ∧ ... ∧ Pₖ) is valid.

Step 4: By De Morgan's law: ¬(P₁ ∧ ... ∧ Pₖ) ≡ ¬P₁ ∨ ¬P₂ ∨ ... ∨ ¬Pₖ. So S = ¬P₁ ∨ ¬P₂ ∨ ... ∨ ¬Pₖ.

Step 5: Verification: the set {P₁, ..., Pₖ} is inconsistent (no assignment makes all true) iff ¬P₁ ∨ ... ∨ ¬Pₖ is always true (valid) — which makes sense because in every assignment, at least one Pᵢ must be false.

Key Insight: Consistency is equivalent to the satisfiability of the conjunction. This bridges logical consistency with truth assignments.

Set 4 Medium — Consistency Check Practice (Set 1)▾

Determine whether each set of formulas is consistent (propositional logic):

(a) {p → q, q → r, p ∧ ¬r}

(b) {p ∨ q, ¬p, ¬q}

(c) {p → q, q → ¬p, p}

Solutions

(a): From p ∧ ¬r we have p true and r false. p → q forces q true. q → r forces r true (since q is true). But we already have r false. Contradiction! The set is inconsistent.

(b): From ¬p and ¬q, both p and q are false. Then p ∨ q is false. So no assignment satisfies all three. The set is inconsistent.

(c): From p we have p true. p → q forces q true. q → ¬p gives ¬p (since q is true), so p is false. But p is true. Contradiction! The set is inconsistent.

Key Insight: To check consistency, try to find a satisfying assignment. If the formulas force a contradiction, the set is inconsistent.

Set 4 Web / Extra — Consistency Check Practice (Set 2 — Extra)▾

Determine whether each set is consistent:

(a) {p ∨ q, ¬p, r → q, ¬r}

(b) {p → q, ¬q → r, p, ¬r}

(c) {p ↔ q, q ↔ r, p ↔ ¬r}

(d) {p ∧ q, p → ¬q}

(e) {∀x (P(x) → Q(x)), ∃x P(x), ∀x ¬Q(x)}

Solutions

(a): From ¬p, p is false. Since p ∨ q and ¬p, q must be true. r → q is automatically true when q is true (regardless of r). ¬r means r is false. So assignment: p=F, q=T, r=F satisfies all four. The set is consistent.

(b): From p and p → q, we get q true. ¬q → r: since q is true, ¬q is false, so ¬q → r is vacuously true. ¬r means r false. So assignment: p=T, q=T, r=F satisfies all four. Consistent.

(c): From p ↔ q and q ↔ r, we have p = q = r. But p ↔ ¬r requires p = ¬r. Since p = r (from the first two), this forces p = ¬p, which is impossible. The set is inconsistent.

(d): From p ∧ q, p is true and q is true. Then p → ¬q becomes T → F = F. So the last formula is false. No assignment can satisfy both. The set is inconsistent.

(e): From ∀x ¬Q(x), all Q(x) are false. From ∃x P(x), there is some a with P(a) true. But ∀x (P(x) → Q(x)) says P(a) → Q(a), which is T → F = F. Contradiction! The set is inconsistent.

Key Insight: Consistency in predicate logic works the same way as propositional logic — find a model (domain + assignment) satisfying all formulas. The challenge is that domains can be infinite, so inconsistency may require a proof rather than a truth table.

Common Mistake

Mistake: Thinking vacuously true implications always make a set consistent.

Why it fails: A vacuous implication (false antecedent) doesn't help satisfy other formulas. Always check whether ALL formulas can be simultaneously true, not just some of them.

Set 4 Completeness▾
Set 4 Definition & Key Facts▾
Completeness (LS01): A set of axioms is complete when every true proposition can be proved from those axioms.
Propositional Logic is Complete: Every valid propositional formula (tautology) has a proof using natural deduction or truth tables. This is the Completeness Theorem for Propositional Logic.
First-Order Logic is Complete (Gödel 1929): In his doctoral dissertation, Gödel proved that first-order logic is complete — every logically valid first-order sentence has a proof. This is Gödel's Completeness Theorem (different from his Incompleteness Theorems).
But Arithmetic is Not Complete: While first-order logic itself is complete, any first-order axiomatization of arithmetic (like Peano Arithmetic) cannot be complete — this is Gödel's Incompleteness Theorem.

Key distinction: Completeness of a logic (every valid formula is provable) vs completeness of an axiom system (every true statement is provable). First-order logic is complete; Peano arithmetic is not.

Consistency: A set of axioms is consistent when you cannot prove that False is True.
Completeness: A set of axioms is complete when every True proposition can be proved from those axioms.
Gödel's Incompleteness Theorem: A set of axioms cannot be both consistent and complete. (LS01)
Why This Matters

Step 1: Given a set of axioms that has consistency (because we need it — a contradictory system proves everything and is useless), there will be True statements that cannot be proved.

Step 2: You cannot disprove them, but to prove them you need more axioms.

Step 3: But if you add more axioms, that bigger set of axioms will have other statements that cannot be proved. The incompleteness is inescapable — no finite (or recursively enumerable) set of axioms can capture all truths of arithmetic.

Q1: Is propositional logic consistent? Is it complete? Explain.

Solution

Answer: Yes on both counts. Propositional logic is consistent (there is no formula P such that both P and ¬P are provable — truth tables can verify this). It is also complete: for every valid formula (tautology), there exists a proof (e.g., using natural deduction or truth tables). This is the Soundness and Completeness Theorem for propositional logic.

Key Insight: Propositional logic is "weak enough" to be fully understood — every question can be decided by truth tables. Adding quantifiers and arithmetic makes the system more expressive but introduces incompleteness.

Q2: What is the difference between Gödel's Completeness Theorem (1929) and his Incompleteness Theorems (1931)?

Solution

Step 1 — Completeness Theorem (1929): Gödel proved that first-order predicate logic is complete — every logically valid sentence has a proof. This was his doctoral dissertation and is a positive result: first-order logic captures all logical truth.

Step 2 — Incompleteness Theorems (1931): Two years later, Gödel proved that any consistent axiomatization of arithmetic is incomplete — there are arithmetic truths that cannot be proved from the axioms. This is a negative result about arithmetic, not about pure logic.

Step 3 — The relationship: The Completeness Theorem says "what is logically true is provable." The Incompleteness Theorems say "what is arithmetically true is not the same as what is provable from any fixed set of axioms." Logic itself is complete; arithmetic axiomatizations are not.

Key Insight: Don't confuse the two! Gödel's Completeness Theorem is about logical validity. Gödel's Incompleteness Theorems are about arithmetic truth. They sound contradictory but address different things.

Q3: True or False — "A complete system guarantees that every true statement has a proof." Is this always a desirable property?

Solution

Answer: True by definition — that's exactly what completeness means. However, completeness may require the system to be very weak. For example, Presburger arithmetic (natural numbers with addition only, no multiplication) is both consistent and complete — but it cannot express multiplication, severely limiting what you can say.

Key Insight: Completeness is a trade-off with expressiveness. The more you can say, the harder it is to have a complete axiomatization. Gödel showed that once you can express enough arithmetic (multiplication), completeness becomes impossible.

Set 4 Gödel's Incompleteness Theorems▾
Set 4 Statement of the Theorems▾
First Incompleteness Theorem (Gödel 1931): Any consistent formal system F within which a certain amount of elementary arithmetic can be carried out is incomplete — there are statements of the language of F which can neither be proved nor disproved in F. (M4CS §8.4)
Second Incompleteness Theorem (Gödel 1931): For such a system F, the consistency of F cannot be proved within F itself. In other words, no consistent system is strong enough to verify itself. (M4CS §8.4)
In Plain English

First Theorem: If math is consistent (no contradictions), then there are true mathematical statements that can never be proven.

Second Theorem: Math cannot prove its own consistency. We must accept consistency on faith or from a more powerful system.

Implication for ZFC: The ZFC axioms are not sufficient to settle all questions — for example, the Continuum Hypothesis is independent of ZFC (neither provable nor disprovable). This was shown by Gödel (consistency with ZFC) and Paul Cohen (independence from ZFC). (M4CS §8.4)
Set 4 The Self-Reference Mechanism (Diagonalization)▾
Core Idea — The Liar Paradox Variant: Gödel constructed a statement G that says "G is not provable in system F." If G is provable, then G says it's unprovable (contradiction). If ¬G is provable, then G is provable (contradiction, assuming consistency). Thus G is true but unprovable.
How the Self-Reference Works

Step 1 — Gödel Numbering: Every symbol, formula, and proof in the formal system is assigned a unique natural number (a Gödel number). This turns statements about proofs into statements about numbers.

Step 2 — The Proof Predicate: Define a formula Prov(x) meaning "the formula with Gödel number x has a proof in the system." This is an arithmetic property of the number x.

Step 3 — The Diagonal Lemma: For any formula φ(x) with one free variable, there exists a sentence D such that D ↔ φ(⌜D⌝) — D says "I have property φ." This is the self-reference construction, analogous to Cantor's diagonalization: list all formulas with one free variable, then construct a formula that differs from each at the "diagonal" position.

Step 4 — The Gödel Sentence: Apply the Diagonal Lemma to ¬Prov(x): obtain G such that G ↔ ¬Prov(⌜G⌝). G says "I am not provable."

Step 5 — Undecidability: If G is provable, then Prov(⌜G⌝) is true, but G says ¬Prov(⌜G⌝) — contradiction (assuming soundness). If ¬G is provable, then ¬Prov(⌜G⌝) is false, so G is true and provable — contradiction (assuming consistency). Hence neither G nor ¬G is provable, yet G is true.

Key Insight: The self-reference is achieved through arithmetic — the system is talking about itself through numbers, not through direct self-reference. This avoids the informal "liar paradox" while achieving the same effect.

Common Mistake

Mistake: Thinking Gödel's theorem means "some truths are forever unknowable."

Clarification: It means some truths are unprovable within a given formal system. By moving to a more powerful system (e.g., adding axioms), we can prove those statements — but the new system will have its own unprovable truths. The incompleteness is about the limits of formal systems, not the limits of human understanding.

Q1: Explain why adding new axioms to a system does not "solve" the incompleteness problem.

Solution

Step 1: Suppose we have a consistent system F with an unprovable truth G. We add G as a new axiom to get F'.

Step 2: The First Incompleteness Theorem applies to F' as well — because F' still encodes arithmetic. There will be a new statement G' that is true but unprovable in F'.

Step 3: This process never ends: any consistent extension that is recursively axiomatizable is still incomplete. Incompleteness is a permanent feature of any formal system strong enough to express arithmetic.

Key Insight: Gödel's theorem is not about a specific axiom set — it's about any possible axiom set. The incompleteness is inescapable.

Q2: Why does the Second Incompleteness Theorem matter for computer science?

Solution

Step 1: If we write a program that proves theorems of arithmetic (a theorem prover), the Second Theorem tells us that the program cannot prove its own correctness (consistency) — unless it's inconsistent, which is worse.

Step 2: This means formal verification of complex systems has inherent limits. A sufficiently powerful system cannot fully verify itself.

Key Insight: This is why we need external verification, human reasoning, and multiple levels of checking in critical software systems.

Q3: What is the relationship between the Liar Paradox and Gödel's sentence?

Solution

Step 1: The Liar Paradox ("This statement is false") creates a paradox because it self-referentially claims falsehood — if true then false, if false then true.

Step 2: Gödel's sentence ("This statement is unprovable") avoids direct paradox because it claims unprovability, not falsehood. If it's false, then it is provable (which could be true) — there's no immediate contradiction.

Step 3: The resolution: if the system is consistent, G must be true but unprovable. If the system is inconsistent, G is provable (but then everything is provable).

Key Insight: Gödel replaced "false" (which creates a paradox) with "unprovable" (which creates incompleteness, not paradox). This subtle shift is what makes the theorem work.

Solution

Answer: No. Gödel's theorem shows that there are limits to what any single formal system can prove — not that mathematics itself is unreliable. Most mathematical work happens in systems far weaker than full arithmetic, and these are perfectly well-understood.

Solution

Q4: Explain how Cantor's diagonalization argument is similar to Gödel's construction.

Solution

Step 1 — Cantor: List all real numbers between 0 and 1 in decimal form. Construct a new number whose n-th digit differs from the n-th digit of the n-th number on the list. This new number differs from every number on the list — so the list is incomplete.

Step 2 — Gödel: List all formulas with one free variable. The Diagonal Lemma constructs a formula that says "I have property ¬Prov(x)" — differing from each formula at the "diagonal" position (its own Gödel number). This statement is true but not on the list of provable formulas.

Step 3 — The pattern: Both arguments assume a complete enumeration, then construct an element that differs from every element at the diagonal position. Cantor shows the reals are uncountable; Gödel shows provable truths don't exhaust all truths.

Key Insight: Diagonalization is the most important proof technique you may never explicitly study — it underlies Cantor (uncountability), Gödel (incompleteness), and Turing (undecidability).

Q5: Why does the First Incompleteness Theorem require the system to be "consistent"? What happens if the system is inconsistent?

Solution

Step 1: The proof assumes consistency to reach the conclusion that G is unprovable but true. If the system is inconsistent, the Principle of Explosion applies — from a contradiction, anything is provable, including G and ¬G.

Step 2: In an inconsistent system, the Gödel sentence G would be provable (everything is provable), but the system would still be "complete" in the trivial sense — every statement is provable, including false ones. This is worse than incompleteness.

Step 3: So consistency is a necessary premise for the interesting conclusion (true but unprovable statements exist). An inconsistent system proves everything — but is useless because it proves falsehoods too.

Key Insight: Consistency is like the "ground truth" of a system. Without it, the system is worthless. With it, incompleteness is inevitable for sufficiently powerful systems.
